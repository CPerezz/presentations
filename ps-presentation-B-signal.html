01 CONCEPT / 02 HOW IT WORKS

Sync, Don't Execute

  1. Snap sync the full account trie (~50 GB)
  2. Snap sync selected contract storage tries (~9 GB for USDC + DAI)
  3. Process Block Access Lists from pivot to HEAD
  4. Follow tip: apply BAL diffs per block. No transaction re-execution.

BALs (EIP-7928) record ALL state changes per block — consensus-enforced completeness

04
01 / 02 / 03 / 04 ENABLES

Inclusion Lists Need State

  • FOCIL (EIP-7805): 16 includers per slot
  • Includers select transactions from the mempool
  • Valid txs = harder for builders to exclude
  • Account data makes includers EFFECTIVE
  • PS drops hardware from 640 GB to 50–80 GB

Without PS in a post-ZKEVM world, only builders and RPC providers can be effective includers
