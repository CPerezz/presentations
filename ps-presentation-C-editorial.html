I. The Present

Where We Are

Today, Ethereum's state is one of its strongest defensive assets.

  • ~640 GB, replicated across 10,000+ validators
  • Every validator re-executes every transaction
  • Geographic distribution across the globe
  • Result: censorship requires targeting the entire planet
State replication is Ethereum's immune system.
2 / 16

I. The Present

Where We're Going

ZKEVMs will fundamentally change the validator's relationship to state.

  • A SNARK proof replaces re-execution
  • Verification in milliseconds, without holding state
  • Validators become rational: if they don't need state, they won't hold it
  • The invisible hand pushes state out of validator machines
3 / 16

II. The Problem

The World Without Incentives

Without incentives to hold state, replication collapses.

  • 10,000 validators → ~50 operators (builders + RPC providers)
  • State becomes centralized infrastructure, not distributed commons
  • FOCIL inclusion lists require state to find transactions
  • Censorship scenario: cut off state serving for targeted addresses → invisible, protocol-level censorship

State expiry was proposed as a solution. It adds protocol complexity without solving centralization — wallets still need the same providers for expired state.

III. Partial Statefulness

The Elegant Insight

Each contract's storage is an independent subtrie.

  • The account trie stays complete — you can prove any account
  • Tracked contracts: you hold their full storage trie, can generate proofs
  • Untracked contracts: you keep only the storage root hash
  • The MPT topology lets you skip entire subtries without breaking integrity

III. Partial Statefulness

Following the Chain

BALs (EIP-7928, Glamsterdam) make partial statefulness possible.

  • BALs record all state changes per block — consensus-enforced completeness
  • Apply diffs directly: no transaction re-execution
  • Security: state root recomputed from diffs + sync committee signatures (pre-ZKEVM) or SNARK verification (post-ZKEVM)
Stronger than a light client: you recompute the state root, not just trust the header.

IV. What This Enables

Censorship Resistance

FOCIL needs state. PS provides it.

  • EIP-7805: 16 includers per slot select transactions from mempool
  • Account data makes includers effective — valid txs are harder for builders to exclude
  • PS drops the hardware bar from ~640 GB to ~50–80 GB
  • Without PS, post-ZKEVM includers are only builders and RPC providers
Account data makes includers effective, not just eligible.

IV. What This Enables

State as a Market

Partial statefulness creates a natural market for state serving.

  • Emergent, out-of-protocol — different operators, different strategies
  • Hot state = cheap (many holders), cold state = premium (few holders)
  • Load balancers route queries to PS nodes tracking relevant contracts

The attestation analogy: one query ≈ nothing. A year of queries ≈ meaningful revenue.

IV. What This Enables

Natural Redundancy

The market creates a redundancy gradient.

  • USDC: replicated by thousands. ENS: by dozens. Obscure contracts: by a few.
  • No protocol changes needed — market handles stratification
  • PS achieves what state expiry promises: hot state distributed, cold state held by specialists
  • Users never experience "account expired" — just fewer replicas for cold state
This is state expiry through market forces, not protocol mandates.

The Validator Floor

  • Validity-Only Partial Statelessness: just the account quadruplets
  • Enables: mempool management + FOCIL inclusion + SNARK verification
  • Any PS node is also a VOPS node
The minimal cost of keeping Ethereum censorship-resistant in a post-ZKEVM world.

V. The Consumers

Agents as Consumers

AI agents are the first Ethereum users that are both autonomous and trust-sensitive.

  • Unlike humans who trust MetaMask + Infura, agents managing capital need verifiable state
  • PS nodes serve proof-carrying responses — cryptographically verified, not trusted
  • Domain specialization aligns naturally: DeFi agent ↔ DeFi PS node

The ZKEVM transition creates the need. Agents become PS's most natural consumer.

VI. Open Design Space

On the Inseparable Graph

A fair critique: DeFi state is deeply interconnected.

When valid

Large aggregators need the full graph. MEV searchers need complete state.

When not

VOPS nodes. ENS. Privacy protocols. Specialized balance queries.

PS nodes don't serve everything. They serve one piece. Different nodes serve different pieces. Proofs batch across nodes sharing the same state root.

This is not about replacing full nodes. It's about creating a spectrum of participation.
